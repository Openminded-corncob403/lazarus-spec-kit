# Firebird Database — Cursor Rules

Use estas regras ao desenvolver com banco de dados Firebird via SQLdb ou ZeosLib no Lazarus.

## Conexão Obrigatória

```pascal
{ SEMPRE configurar em SQLdb: }
FConnection.ConnectorType := 'Firebird';
FConnection.CharSet := 'UTF8';
FConnection.Transaction := FTransaction;
```

## Generators e Auto-Increment

| Abordagem | Versão | Exemplo |
|-----------|--------|---------|
| `GEN_ID(generator, 1)` | Todas | Clássico com trigger BI |
| `NEXT VALUE FOR seq` | FB 3+ | Sintaxe SQL padrão |
| `IDENTITY` | FB 3+ | `GENERATED BY DEFAULT AS IDENTITY` |

## Execução e RETURNING

```pascal
{ Usar ExecSQL para operações DML simples }
LQuery.SQL.Text := 'UPDATE customers SET status = 1 WHERE id = :id';
LQuery.ExecSQL;

{ Usar Open para RETURNING (SQLdb/Zeos geralmente exigem Open se retornar dados) }
LQuery.SQL.Text := 'INSERT INTO ... RETURNING id';
LQuery.Open;
LId := LQuery.Fields[0].AsInteger;
```

## Stored Procedures

| Tipo | SUSPEND | Chamada (Lazarus) |
|------|---------|----------------|
| **Selectable** (retorna dados) | Sim | `SELECT * FROM SP_NOME(...)` + `Open` |
| **Executable** (ação) | Não | `EXECUTE PROCEDURE SP_NOME(...)` + `ExecSQL` |

## Transactions

```pascal
{ Transação explícita para operações compostas }
FTransaction.StartTransaction;
try
  FRepoA.Insert(LObjA);
  FRepoB.Insert(LObjB);
  FTransaction.Commit;
except
  FTransaction.Rollback;
  raise;
end;
```

## Tratamento de Erros Firebird

```pascal
{ Verifique a exception da engine de dados usada (ex: EDatabaseError em SQLdb ou EZSQLException em Zeos) }
except
  on E: EDatabaseError do
  begin
    // Tratamento genérico ou análise de E.Message para identificar a violação (FK, UK)
    raise EConflictException.Create('Falha no banco de dados: ' + E.Message);
  end;
end;
```

## Tipos de Dados — Cuidados

| Firebird | Cuidado |
|----------|---------|
| `DATE` (Dialect 3) | Apenas data, sem hora — use `TIMESTAMP` para data+hora |
| `CHAR(N)` | Preenche com espaços — prefira `VARCHAR(N)` |
| `BLOB SUB_TYPE TEXT` | CLOB — usar campos texto equivalentes |
| `NUMERIC(P,S)` | Exato — usar para valores monetários |
| `BOOLEAN` | Apenas FB 3+ — usar `SMALLINT CHECK (VALUE IN (0,1))` para FB 2.5 |

## Domains — Tipos Reutilizáveis

```sql
CREATE DOMAIN DM_ID     AS INTEGER NOT NULL;
CREATE DOMAIN DM_NAME   AS VARCHAR(100) NOT NULL;
CREATE DOMAIN DM_MONEY  AS NUMERIC(15,2) DEFAULT 0;
CREATE DOMAIN DM_STATUS AS SMALLINT DEFAULT 0 CHECK (VALUE BETWEEN 0 AND 2);
```

## Proibições em Firebird

- ❌ Dialect 1 — usar SEMPRE Dialect 3
- ❌ Concatenar SQL — usar parâmetros
- ❌ Ignorar `CharacterSet` — definir `UTF8` sempre
- ❌ `PAGE_SIZE 4096` — usar 16384 para produção
- ❌ Queries sem índice em colunas de WHERE/JOIN
- ❌ `CREATE TABLE IF NOT EXISTS` — não existe no Firebird (verificar metadata)

## Metadata — Verificar Existência

```pascal
{ Tabela existe? }
SELECT COUNT(*) FROM RDB$RELATIONS
WHERE RDB$RELATION_NAME = 'CUSTOMERS' AND RDB$SYSTEM_FLAG = 0;

{ Coluna existe? }
SELECT COUNT(*) FROM RDB$RELATION_FIELDS
WHERE RDB$RELATION_NAME = 'CUSTOMERS' AND RDB$FIELD_NAME = 'EMAIL';
```
